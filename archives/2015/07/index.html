RegExp对象

除了RegExp构造函数之外,RegExp对象还支持三个方法和一些属性。

RegExp属性

source：只读字符串，正则表达式的文本
global：只读布尔值，是否是全局检索
ignoreCase：只读布尔值，是否忽略大小写
multiLine：只读布尔值，是否多行匹配
lastIndex：可读写的数字，如果匹配模式带有g，则存储的是整个串下一次检索的开始位置

Javascript正则表达式二

用以模式匹配的String方法

search()

它的参数是一个正则表达式，返回第一个与之匹配的子串的起始位置，如果找不到与之匹配的子串，则返回-1
如果search的参数不是正则表达式，则首先会通过RegExp的构造函数将它转换成正则表达式。search不支持全局检索，以为他忽略正则表达式参数的g

replace()

用以执行检索和替换操作，其中第一个参数是一个正则表达式，第二个参数是要进行替换的字符串。如果不带修饰符g则只替换第一个与之匹配的子串。如果repalce的第一个参数是一个字符串不是正则表达式，则会直接进行替换。

match()

它唯一的一个参数是正则表达式（或通过RegExp转换成正则表达式），返回的是一个有匹配结果组成的数组。如果这个正则表达式没有指定g，则match不会进行全局检索。但即使不是全局匹配，返回的也会是一个数组。

split()

Javascript正则表达式一

正则表达式是一个描述字符模式的对象。我们主要利用正则表达式来完成强大的模式匹配和文件检索与替换功能。Javascript中的正则表达式用RegExp对象来表示。不过更建议通过直接量语法来创建。正则表达式直接量定义为包含在//之间的字符串。

例如：

var pattern = /dongmei.yang/

匹配的是包含dongmei.yang的字符串

直接量字符

正则表达式中所有的字母和数字都是按照字面含义进行匹配的。javascript正则表达式也支持非字母的字符匹配，这些字符需要通过反斜线\作为前缀进行转义。

字符匹配
\oNUL字符
\t制表符
\n换行符
\v垂直制表符
\f换页符
\r回车符
\xnn由十六进制数nn指定的拉丁字符
\uxxxx由十六进制数xxxx指定的unicode字符
\cX控制字符^X

字符类

将直接量字符单独放进方括号[]就组成了字符类。一个字符类可以匹配他所包含的任意字符。

var pattern = /[a,b,c]/;

匹配的就是a,b,c的任意字符。另外可以通过^来定义否定类（将^作为左方括号的第一个字符），它用来匹配任何不包含在方括号内的字符。例如：

var pattern = /[^a,b,c]/;

匹配的是除了a,b,c之外的任意字符。字符类可以使用连字符来表示字符范围，比如[0-9]表示数字,[a-z]表示小写字母
正则表达式的字符类

字符匹配
\w任何ASCII组成的单词
\W任何不是ASCII组成的单词
\s任何Unicode空白符
\S任何非Unicode空白符
\d任何ASCII数字，等价于[0-9]
\S除了ASCII数字之外的任何字符
\S任何非Unicode空白符

在方括号内也可以写这些转义字符

重复

正则表达式的重复字符语法

字符匹配
{m,n}匹配前一项至少m次，但不能超过n次
{m,}匹配前一项至少m次
{m}匹配前一项m次
? 匹配前一项0次或1次
+ 匹配前一项1次或多次
* 匹配前一项0次或多次

非贪婪重复

之前列的匹配重复字符是尽可能多的匹配，而且允许后续的正则表达式继续匹配，这种匹配方式成为贪婪的匹配，同样可以使用正则表达式进行非贪婪匹配。只需在待匹配的字符后面跟随一个问号即可：??, +?, *? 或{1, 5}? 使用非贪婪匹配所得到的结果可能和期望的并不一致。/a+?b/匹配尽可能少的a和一个b，当它用来匹配aaab时，可能期望的结果是最后一个a和b，但是结果并非如此，匹配的是整个串。这是因为正则表达式的模式匹配总是会寻找字符串中第一个可能匹配的位置。

选择、分组和引用

正则表达式的语法还包括指定选择项、子表达式和引用前一子表达式的特殊字符。字符|用于分割供选择的字符。例如，ab|cd|ef，可以匹配ab也可以匹配cd也可以匹配ef。选择项的匹配时从左至右，知道发现了匹配的项。

正则表达式的圆括号有多种作用：
1.把单独的项组合成子表达式 /java(script)?/
2.在完整的模式中定义子模式 /[a-z]+(/d)/
3.允许在同一正则表达式后面引用前面的子表达式，这是通过在\后面加一位或多位数字来实现的

指定匹配位置

正则表达式中的多个字符才能匹配字符串的一个字符。任意正则表达式都可以作为锚点条件，如果在符号(?=和)之间加入一个表达式，他就是一个先行断言，用以说明圆括号里面的内容必须正确匹配。带有(?!的是负向先行断言，用已指定接下的字符都不必匹配。

jquery prop 与 attr的区别

区别

在高版本的jquery引入prop方法后，什么时候该用prop？什么时候用attr？它们两个之间有什么区别？这些问题就出现了。

关于他们两个的区别，进行了一下简单的总结：

  1. 对于HTML元素本身就带有的固有属性，在处理时，使用prop方法。
  2. 对于HTML元素我们自己自定义的DOM属性，在处理时，使用attr方法。

建议使用prop的两种情况：

  1. 值为true/false的属性,建议使用prop, 例如checked selected
  2. 对于一些和路径相关的属性，两者取得值也不尽相同，但是同样attribute取得是字面量，property取得是计算后的完整路径

值为true/false的情况

<input id="chk1" type="checkbox" />是否可见
<input id="chk2" type="checkbox" checked="checked" />是否可见

$('#chk1').prop('checked') == false
$('#chk2').prop('checked') == true

如果上面使用attr方法，则会出现：

$('#chk1').attr('checked') == undefined
$('#chk2').attr('checked') == 'checked'
